API reference#

pgsc_calc has two main use cases:

  • A bioinformatician or data scientist wants to calculate some polygenic scores using an Unixy operating system and a terminal

  • A normal person (e.g. a biologist or other researcher) wants to calculate some polygenic scores using a web browser

To simplify the second use case, the workflow is designed to be launched programmatically on a private cloud using an API. API parameters are specified using JSON. The web platform is still under development.

Specifying target genomes with JSON#

[
    {
        "sample": "example",
        "vcf_path": "/path/to/genome.vcf.gz",
        "chrom": 22
    }
]

Specifying workflow parameters with JSON#

Complete API call#

{
    "target_genomes": [
        {
            "sample": "example",
            "vcf_path": "/path/to/genome.vcf.gz",
            "chrom": 22
        }
    ],
    "nxf_params_file": {
        "scorefile": "/path/to/scorefile.txt",
        "format": "json"
    },
    "nxf_work": "/workspace/unique_work_directory/",
    "id": "unique_id"
}

Implementation details#

The API is designed using an event-driven approach with Argo Events. Briefly, a sensor constantly listens on a Kubernetes cluster for Kafka messages to launch the pipeline. Once the message is received, a nextflow driver pod is created and the workflow is executed using the K8S executor. The status of the workflow instance is reported using Nextflow’s weblog and a second sensor. We didn’t have Nextflow Tower at the time.
